Problem: Counting Rectangles is Fun!

Problem statement Given a boolean matrix and sub-matrix queries. A "good" sub matrix is one which has all 0s. For each query, find how many good sub matrix it contains?

At a first glance (and with my fear with 2D-query problems) this looked like some 2D-segment tree to me. Later after spending sometime realized that this could be solved with the famous same old prefix sum technique. I used standard way to calculate submatrix sum with some modifications(Technique-4). What motivated me to write this blog and a 3b1b style animation(cost me a Sunday to learn manim) is learning technique-1. Interestingly, while I was skimming through the solutions(may be I should do this more often as a practice), I found that most people solved the problem with one of the four techniques explained below. I only provide a brief few lines gist of the algorithms as this blog is only meant for someone who got interested in the problem decides to solve the problem with little hints.

  • Technique-1(Foreward DP): Check tourist's solution

    Usually while searching for solutions, I sort codeforces's solution list by time of submission to see how the coders who implemented quickly thought of the problem i.e whether they are using some standard approach that might generalize well with other problems. Here, in this case(no prizes for guessing) the very first solution was from the "tourist". Although the implementation was quite clean the idea did not hit me. After spending some time on the foreward dp transformations, I realized the nice geometrical idea involved. Any "good" rectangle is only once added to all the covering rectangles after 4 stages of iterations.

  • Technique-2(DP sets DP+Inclusion/Exclusion)

    Basic idea of this approach is to calculate the query in two steps. In first step, $dp_1[i][j][k][l]$ counts all good rectangles in $rect(i,j,k,l)$ with top-left corner $(i,k)$ fixed and bottom right corner $(x,y)$ such that $x<=k \space\&\space y<=l$. In the second iteration, use $dp_2$ to find all good rectangles which end at (x,y) such that $x<=k \space\&\space y<=l$ and start after $(i,k)$. $dp_1$ is needed by $dp_2$ to find the unknown set i.e all good rectangles starting at $(i,k)$ ending anywhere before $(j,l)$.

    $$ Complexity: O(n^4)$$

  • Technique-3 (Inclusion-Exclusion DP)

    Idea: Use inclusion exclusion to find the answer to $dp(i,j,k,l)=dp(i+1,j,k,l) \cup dp(i,j-1,k,l) \cup dp(i,j,k+1,l) \cup dp(i,j,k,l-1)$.

  • Technique-4 (Prefix Sum DP)

    Idea behind this is simple. Calculate $dp(i,j,k,l)$ using $dp(i,j,k-1,l) \cup dp(i,j,k,l-1)$ and calculate all those good rectangles that end its right corner in $(j,l)$. Calculating the last part becomes easy if we calculate all the zero-towers (col-wise) from a particular cell.

    $$ Complexity: O(n^5) $$
